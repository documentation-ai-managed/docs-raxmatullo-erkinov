## Overview

Raxmatullo Erkinov provides a centralized space to organize, manage, and collaborate on your project documentation. You create structured docs, customize layouts, and keep everything version-controlled without complexity. Start building your knowledge base today.

<Callout kind="info">
  Raxmatullo Erkinov supports Markdown, MDX components, and integrates seamlessly with your Git workflow.
</Callout>

## Quick Start

Get up and running in minutes. Follow these steps to create your first documentation page.

<Steps>
  <Step title="Create Account" icon="user-plus">
    Sign up at `https://dashboard.example.com/register` with your email.
  </Step>
  <Step title="New Project" icon="plus">
    Click "New Project" and name it, e.g., "My Project Docs".
  </Step>
  <Step title="Add First Page" icon="file-plus">
    Use the editor to write your `introduction.mdx`. Here's a starter:

````mdx
```javascript
// Example script for your docs
console.log("Welcome to Raxmatullo Erkinov!");
```
````
  </Step>
  <Step title="Publish" icon="globe">
    Hit "Publish" to make your docs live at `https://docs.example.com/my-project`.
  </Step>
</Steps>

## Brand Guidelines

Follow these guidelines to maintain consistency across your documentation.

<Expandable title="Primary Brand Color" default-open="true">
  Use `#3B82F6` as your primary blue for headings, links, and accents. It represents trust and professionalism.

  Example CSS:

````css
:root {
  --brand-primary: #3B82F6;
}

a {
  color: var(--brand-primary);
}
````
</Expandable>

<Callout kind="tip">
  Always test your theme with the brand color on different devices for optimal readability.
</Callout>
