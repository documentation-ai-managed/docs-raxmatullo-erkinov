## Webhook Setup

Set up webhooks to trigger actions on documentation events like page updates or new versions.

<Steps>
  <Step title="Create Webhook" icon="plus">
    Navigate to <kbd>Settings</kbd> {">"} <kbd>Integrations</kbd> {">"} <kbd>Webhooks</kbd>.
    
    Click "New Webhook" and enter your endpoint URL.
  </Step>
  <Step title="Configure Events" icon="settings">
    Select events: `page.updated`, `page.published`, `comment.added`.
    
    Add optional secret for signature verification.
  </Step>
  <Step title="Test and Save" icon="check-circle">
    Send a test payload, then save. Monitor deliveries in the dashboard.
  </Step>
</Steps>

Handle incoming webhooks securely. Verify signatures using the shared secret.

<CodeGroup tabs="Node.js,Python">
  ```javascript
  const crypto = require('crypto');

  app.post('/webhook', (req, res) => {
    const signature = req.headers['x-raxmatullo-signature'];
    const payload = JSON.stringify(req.body);
    const expected = 'sha256=' + crypto.createHmac('sha256', 'YOUR_WEBHOOK_SECRET')
      .update(payload).digest('hex');

    if (signature === expected) {
      console.log('Event:', req.body.event);
      res.status(200).send('OK');
    } else {
      res.status(401).send('Unauthorized');
    }
  });
  ```
  ```python
  import hmac
  import hashlib
  from flask import Flask, request

  app = Flask(__name__)
  SECRET = b'YOUR_WEBHOOK_SECRET'

  @app.route('/webhook', methods=['POST'])
  def webhook():
      signature = request.headers.get('X-Raxmatullo-Signature')
      payload = request.get_data()
      expected = 'sha256=' + hmac.new(
          SECRET, payload, hashlib.sha256
      ).hexdigest()

      if signature == expected:
          print('Event:', request.json['event'])
          return 'OK', 200
      return 'Unauthorized', 401
  ```
</CodeGroup>

## API Access for Custom Extensions

Build custom integrations using the REST API at `https://api.example.com/v1`.

<ParamField header="Authorization" param-type="string" required="true">
  Bearer `{YOUR_API_KEY}`.
</ParamField>

<ParamField query="space_id" param-type="string" required="true">
  Your documentation space identifier.
</ParamField>

Test the API with this endpoint:

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  "https://api.example.com/v1/spaces/{SPACE_ID}/pages"
```

<Response tabs="200">
  ```json
  {
    "pages": [
      {
        "id": "page-123",
        "title": "Introduction",
        "updated_at": "2024-01-15T10:30:00Z"
      }
    ],
    "next_cursor": null

  ```
</Response>
